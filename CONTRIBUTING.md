# Contributing to Java Projects Portfolio

Thank you for considering contributing! This repository is designed as an educational portfolio, and contributions that improve code quality, add features, or enhance documentation are welcome.

## How to Contribute

1. **Fork** the repository.
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Commit your changes**: `git commit -m "Add: brief description"`
4. **Push to the branch**: `git push origin feature/your-feature-name`
5. **Open a Pull Request**.

## Guidelines

- Follow existing code conventions and architecture patterns.
- Level 1 projects use `Makefile` + `javac`. Do NOT add Maven to them.
- Level 2–3 projects use Maven. Do NOT add Makefiles to them.
- Add JUnit 5 tests for any new business logic.
- Update the relevant `README.md` if you add features.
- Keep commits atomic and well-described.

## Ideas for Contributions

- Add JUnit tests to existing services
- Implement JavaFX alternative UIs
- Add password hashing for authentication
- Create Docker Compose setups for MySQL
- Improve error handling and logging
- Add Swagger/OpenAPI documentation

## Code of Conduct

Be respectful. We are all here to learn and build.
